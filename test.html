Frontend guideline

Introduction

How to use this document

This document is a model of the coding standards defined in the project, so you do not need to code everything accordingly.

Please read this agreement before development begins and create coding standards for each project as necessary.

Please manage the coding rules you create with github etc and share it to members.


Purpose of preparing coding convention

  • To keep the code easy to understand
  • To match recognition with members of the project
  • To facilitate handover when members change

Overall Rule

About external files

  • In principle, javascript, CSS etc. are converted to external files

Writing a path

  • In principle, links and images are described by root relative path
  • Because favicon may not display images, describe it by absolute path

Indent

  • Use two halfwidth spaces for both HTML, CSS, and JavaScript

Insert comment

  • Insert comments as needed

Naming convention

Directory name

  • Basically directory names use lowercase letters
  • Make it a name based on English words
  • Together with the notation of snake case or camel case or kebab case


HTML

Basic rules (SEO edition)

  • Disable use of hidden text (hide in CSS) and hidden links (tooltips etc. are fine)
  • Always set the alt attribute for images. (However, alt = "" for images not related to documents)

HTML Basic Rule

  • DOCTYPE is based on "HTML 5"
  • Character code is basic "utf-8"
  • All tags and attributes are written in lower case
  • Enclose attribute values ​​in double quotation marks

W3C compliant

  • Describe W3C Valid description except for implementation reasons

title

  • The page title is described according to the content of the contents of the page
  • Describe in ascending order from the subordinate directory to the site top page

(Example) 

<title> News｜Lab Inc. </title>

description

  • Based on the number of characters displayed in the search result, specify contents of about 120 full-width characters.

mail address

The markup of the mail address describes the encoded one

(Example)

<a href="mailto:test@test.com">test@test.com</a>

CSS

Basic rules

  • important! Not to use except when absolutely necessary

Javascript

Basic rules

  • Definition in global space is not basically done
  • Make it a class name like '.js-hoge' for class used in Javascript

Image

  • Maximize weight reduction without deterioration by using compression tool
  • Do not use online compression services such as TinyPNG except images that may leak

reference:

ImageAlphaYou can reduce the number of colors in the image

jpegtranNpm package that can compress jpg

optipngPng compression file capable npm package


  • Use JPEG, PNG, and GIF properly
Reference: Using images properly When it is necessary to transmit the background → png For images with few colors, such as diagrams, line drawings, etc. → png Pictures, images with gradation, etc., when the number of colors is large → jpg Loading image etc. → gif
  • Make an image read first if necessary.
  • Sprites are created by basic engineers.

reference： http://compass-style.org/


CSS, JS file compression, obfuscation

  • CSS, and JS files are compressed and obfuscated (in the development environment, those who do not compress them are easier to debug, so only the production environment is compressed)
example： gulp-uglify gulp-cssmin

Smartphone site coding

Basic rules

  • When providing content with the same contents as PC version for smartphone, put the rel = "canonical" tag indicating it

General Markup

  • The resolution is based on 1334 x 750, but implements it in a responsive layout or liquid layout so as not to collapse even at terminals with other resolutions

Reference: http://blog.teamtreehouse.com/which-page-layout



viewport

  • Unless otherwise specified, it is as follows 
<meta name="viewport" content="width=device-width, initial-scale=1, minimum-scale=1, maximum-scale=1, user-scalable=no">
  • In order to control the telephone call with numeral link, we also write the following description as necessary 
<meta name="format-detection" content="telephone=no">

Image

  • Specify the image as 1/2 size (Retina compatible)

font size

  • The lower limit of the font size is 10px
